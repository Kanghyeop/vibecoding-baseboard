# 베이스보드 (Baseboard)

기획서 3종(`../001-기획서-클라이언트.md`, `../002-기획서-어드민.md`, `../003-디자인시스템.md`)을 그대로 구현한 베이스라인 게시판 앱.
바이브 코딩 교본용: "기획 문서 3개 → 동작하는 앱"의 본보기.

## 현재 단계 (1단계: 껍데기)

- **로그인은 페이크.** 구글 버튼을 누르면 가상 유저 "졸린 쿠키"(u1)로 즉시 로그인. 어드민도 동일.
- **DB 없음.** 데이터는 서버 인메모리 시드(`src/lib/db.ts` - 유저 5명, 글 10개, 댓글 9개). 서버 재시작하면 리셋.
- **API 통신 구조는 진짜.** 화면은 전부 `/api/*`로만 데이터를 주고받으므로, 나중에 `db.ts`를 Supabase로 갈아끼우면 화면은 그대로 동작.

## 실행

```bash
npm install
npm run dev    # http://localhost:3000
```

## 화면

| 경로 | 화면 |
|---|---|
| `/` | 홈 = 글 목록 (+ 글쓰기 FAB) |
| `/posts/[id]` | 글 상세 + 좋아요 + 댓글 |
| `/write`, `/posts/[id]/edit` | 글 작성/수정 (로그인 필수) |
| `/my` | 마이페이지 (닉네임 수정, 내 글, 로그아웃, 탈퇴) |
| `/admin` | 어드민 - 대시보드/게시글/댓글/유저 (데스크톱 전용) |

- 클라이언트: 360×800 모바일 프레임, 데스크톱에선 가운데 정렬 + 양옆 비움
- 어드민: 프레임 없이 데스크톱 웹 (사이드바 240px + 표)
- 권한 모델: 읽기는 공개, 쓰기(글·댓글·좋아요)는 로그인 - 비로그인 쓰기 시도 시 로그인 모달

## 구조

```
src/
  app/
    (client)/      # 유저 화면 (360×800 프레임 적용)
    admin/         # 어드민 화면 (데스크톱)
    api/           # API 라우트 - 화면과 데이터의 유일한 통로
  components/      # shell(프레임·헤더·탭·로그인모달), icons, admin-ui
  lib/
    db.ts          # 인메모리 데이터 + 시드 ← Supabase 교체 지점
    auth.tsx       # 페이크 로그인 컨텍스트 ← Supabase Auth 교체 지점
    util.ts        # 시간 표기
```

## 디자인 토큰

`003-디자인시스템.md`의 토큰을 `src/app/globals.css`의 Tailwind `@theme`으로 선언.
색은 `bg-point`, `text-faint`, `border-line` 같은 토큰 클래스로만 사용. 임의 hex 금지.
폰트는 Pretendard(CDN).
