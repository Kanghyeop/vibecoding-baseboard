# 베이스보드 - 디자인 시스템

> 목적은 "예쁘게"가 아니라, 모든 화면이 같은 규칙으로 그려져서 어디를 봐도 한 앱처럼 보이게 하는 것.
> 색, 크기, 간격을 매번 고르지 말고 아래 표에서만 꺼내 쓰기. 표에 없는 값이 쓰고 싶어지면 화면이 아니라 이 문서를 먼저 고침.
> 본보기 템플릿: 포인트 컬러 1개만 자기 브랜드색으로 바꾸면 어떤 앱에도 그대로 적용됨.

## 색

기본 발상: 바탕은 흰색, 글자는 검정의 진하기 차이, 색은 포인트 1개뿐.
화면에 색이 보이면 반드시 의미(눌러라, 켜졌다, 위험하다)가 있어야 함.

무채색 (이 7개로 화면의 95%를 칠함)

| 이름 | 값 | 어디에 |
|---|---|---|
| 배경 | `#FFFFFF` | 앱 전체 배경, 헤더, 하단바 |
| 살짝 가라앉는 면 | `#F7F7F8` | 입력창, 카드 배경 |
| 글자-기본 | `#171719` | 제목, 본문 |
| 글자-보조 | `#5A5C63` | 닉네임, 설명 |
| 글자-희미 | `#9EA0A7` | 시간, 자리표시 글, 카운트 |
| 선 | `#E1E2E4` | 테두리, 구분선 |
| 선-약함 | `#F0F0F2` | 목록 행 사이 |

포인트 컬러 (단 1색 - 여기만 자기 브랜드색으로 교체)

| 이름 | 값 | 어디에 |
|---|---|---|
| 포인트 | `#FF5A36` | 주요 버튼, 활성 탭, 눌린 좋아요 |
| 포인트-연함 | `#FFEDE8` | 포인트의 옅은 배경 (활성 칩 등) |

- 한 화면에 포인트색 요소는 1~2개까지만. 전부 강조하면 아무것도 강조 안 됨
- 그 외 의미색: 성공 `#00BF40`, 오류·삭제 `#FF4242`. 이 용도 외엔 안 씀

## 글자

폰트는 Pretendard 하나만. 크기는 아래 6단계만 씀.
중간 크기가 필요해 보이면 크기 말고 굵기나 색으로 차이를 냄.

| 이름 | 크기 | 굵기 | 어디에 |
|---|---|---|---|
| 타이틀 | 22px | bold | 화면 제목 ("홈", "마이") |
| 헤딩 | 19px | bold | 글 상세의 제목 |
| 서브헤딩 | 16px | semibold | 목록의 글 제목, 섹션 제목 |
| 본문 | 15px | regular | 본문, 댓글 |
| 캡션 | 13px | regular | 닉네임, 시간, 카운트 |
| 마이크로 | 11px | regular | "수정됨" 같은 최소 주석 |

- 중요도는 크기로, 강조는 굵기로, 의미는 색으로
- regular보다 얇은 굵기는 금지 (한글이 흐려 보임)

## 간격 · 둥글기

간격은 4의 배수만: 4 / 8 / 12 / 16 / 20 / 32

- 4: 아이콘과 숫자처럼 한 덩어리 안
- 8: 덩어리와 덩어리 사이 (제목과 그 아래 메타 줄)
- 12: 카드·칩 안쪽 여백, 댓글 사이
- 16: 블록 세로 간격
- 20: 화면 좌우 기본 여백 (고정)
- 32: 섹션과 섹션 사이
- 7px, 13px 같은 임의 값 금지

둥글기(모서리)는 3단계 + 원형:

- 8: 작은 칩, 태그
- 12: 입력창, 카드, 버튼
- 16: 안내 창(모달), 아래에서 올라오는 창(바텀시트)
- 원형: 아바타, 글쓰기 동그란 버튼

## 부품 규칙

새 모양을 발명하지 말고 아래 조합으로만 화면을 만듦.

- 주요 버튼: 포인트색 배경 + 흰 글자, 높이 48px. 한 화면에 1개 원칙
- 보조 버튼: 연회색(`#F7F7F8`) 배경 + 검정 글자, 같은 크기
- 입력창: 연회색 배경, 테두리 없음. 누르면(포커스) 포인트색 테두리 1px
- 목록 행: 좌우 여백 20, 위아래 16, 행 사이 약한 선 1px
- 칩(필터): 평소엔 회색 테두리, 선택되면 포인트-연함 배경 + 포인트색 글자
- 아바타: 원형. 목록에선 32px, 상세에선 40px
- 빈 화면: 가운데 정렬로 희미한 글자 한 줄 (+ 필요하면 보조 버튼 1개)
- 확인창: 삭제·차단 전엔 무조건. 확인 버튼만 빨강 허용. 브라우저 기본 창을 쓰지 않고 위 색 표대로 직접 그림
- 토글(스위치): 켜짐은 포인트색, 꺼짐은 비활성 회색. 마케팅 수신 동의 같은 설정용
- 액션시트(⋯ 메뉴): 화면 아래에서 올라오는 선택지 목록. 위에 그랩바(작은 가로 막대), 항목은 가운데 정렬 + 행 사이 약한 선, 되돌리기 어려운 항목(삭제·신고)은 빨강 글자, 맨 아래 회색 취소 버튼. 올라오는 모션 0.2초
- 푸터: 마이페이지 맨 아래. 서비스명 + 약관·방침 링크 + 문의. 전부 작고 희미한 글자, 개인정보처리방침만 굵게

## 화면 틀

반응형은 클라이언트와 어드민이 서로 다른 규칙을 따름. 섞지 않음.

클라이언트 (모바일 앱처럼)

- 기준 화면 360×800. 모바일 앱이라고 가정하고 그림
- 웹에서 봐도 똑같이: 모바일 폭의 세로 기둥을 가운데 두고 양옆은 비워둠. 높이는 브라우저를 꽉 채움. 화면이 늘어나도 내용이 한없이 넓어지지 않음
- 브레이크포인트(화면 폭이 이 기준을 넘으면 모양이 바뀌는 선)는 2구간만:

| 화면 폭 | 기둥 폭 |
|---|---|
| ~1023px (모바일·태블릿) | 풀스크린 (꽉 채움) |
| 1024px~ (데스크톱) | 가운데 480px |
- 상단 헤더: 스크롤해도 고정, 흰 배경, 아래 약한 선 1px. 왼쪽 제목(또는 뒤로가기) + 오른쪽 버튼 1개까지
- 하단 탭바: 메인 탭에서만 보임, 상세·글쓰기에선 숨김. 활성 탭만 포인트색
- 글쓰기 버튼: 우하단 고정, 포인트색 원형 56px
- 구역 나누기: 그림자나 박스 대신 간격(32px)과 1px 선으로. 그림자는 글쓰기 버튼과 안내 창에만

어드민 (회사 어드민답게)

- 모바일 프레임에 구애받지 않음. 일반 웹으로 그림
- 표준 SaaS 어드민 문법: 연회색 페이지 배경 위에 흰색 카드, 목록은 카드 안의 표
- 데스크톱(1024px~): 왼쪽 사이드바 240px 고정 (메뉴 아이콘 + 하단 운영자 정보)
- 모바일(~1023px): 상단 바 + 햄버거 서랍 메뉴, 표는 카드 안에서 가로 스크롤
- 같은 색·글자 규칙을 쓰되 글자만 한 단계 작게(본문 14px)
- 상태 배지는 무채색만: 정상 = 연회색, 숨김·차단 = 검정, 삭제 = 희미+취소선
	- 키컬러는 활성 메뉴와 경고 숫자(미처리 신고)에만

## 하지 않는 것

그라데이션 / 배경색 바꾸기 / 3색 이상 쓰기 / 폰트 섞기 / 임의 px 간격 / 요소마다 다른 둥글기 / 본문 가운데 정렬 / 다크모드(나중에)
