# 베이스보드 - 어드민 기획

> 유저 사이드 기획(001)과 한 쌍. 관리 대상만 자기 아이템으로 바꿔 쓰면 됨.

## 어드민이란

운영자가 글, 댓글, 유저를 들여다보고 정리하는 내부 도구. 제2의 서비스가 아님.
모든 메뉴가 같은 패턴의 반복: 목록 보기 → 하나 클릭 → 처리 버튼 누르기.

## 형태

- 데스크톱 우선이지만 모바일에서도 동작 (반응형)
	- 데스크톱: 왼쪽 사이드바 고정
	- 모바일: 상단 바 + 햄버거 누르면 서랍 메뉴, 표는 가로 스크롤
- 주소는 `/admin`으로 분리. 유저 화면 어디에도 어드민 가는 링크를 두지 않음

## 로그인 · 권한

- 똑같이 구글 로그인
- 단, 허용된 운영자 이메일 명단에 있는 계정만 입장 가능
	- 명단에 없으면 "권한 없음" 화면
	- 명단 관리는 일단 설정 파일 수준으로 (어드민 화면에서 초대하는 기능은 나중에)
- 등급 없음. 명단에 있으면 전부 같은 권한 (슈퍼어드민 안 만듦)
- 누가 언제 뭘 처리했는지(숨김, 삭제, 차단)는 기록으로 남김

## 화면 구조

왼쪽 사이드바 메뉴 5개:

- 대시보드
	- 숫자 5개만: 총 유저 수 / 총 글 수 / 총 댓글 수 / 오늘 새 글 / 미처리 신고
	- 미처리 신고가 1건이라도 있으면 숫자를 포인트색으로 강조
	- 그래프, 기간 비교 없음
- 글 관리
	- 전체 글 목록 (최신순): 제목, 작성자, 시간, 좋아요·댓글 수, 상태(공개/숨김/삭제)
	- 제목이나 닉네임으로 검색
	- 할 수 있는 처리: 숨김 / 숨김 해제 / 삭제
		- 숨김 = 유저 화면에서만 안 보임, 데이터는 그대로
		- 유저가 스스로 지운 글도 어드민에선 보임 (분쟁 대응용)
	- 글 내용을 어드민이 직접 고치는 기능은 없음 (처리는 숨김·삭제까지만)
- 댓글 관리
	- 전체 댓글 목록: 내용 요약, 작성자, 어느 글에 달렸는지, 시간, 상태
	- 처리 규칙은 글과 동일 (숨김 / 삭제)
	- 행에서 해당 글로 바로 이동 가능
- 유저 관리
	- 전체 유저 목록 (가입 최신순): 닉네임, 이메일, 가입일, 쓴 글 수, 상태(정상/차단/탈퇴)
	- 유저 클릭하면 → 그 사람이 쓴 글, 댓글 모아 보기
	- 할 수 있는 처리: 차단 / 차단 해제
		- 차단당하면: 로그인은 되는데 글, 댓글, 좋아요가 전부 막힘. "이용이 제한된 계정입니다" 안내
		- 차단해도 기존 글은 자동으로 숨기지 않음 (필요하면 글 관리에서 개별 처리)
	- 강제 탈퇴는 없음. 차단으로 충분
- 신고 관리
	- 유저가 접수한 신고 목록 (미처리 우선, 최신순): 유형(글/댓글), 대상 내용 요약, 사유, 신고자, 접수일, 상태
	- 대상 내용을 클릭하면 해당 글로 바로 이동
	- 할 수 있는 처리: 대상 숨김 / 기각
		- 대상 숨김 = 신고당한 글·댓글을 숨김 처리하고 신고를 "숨김 처리" 상태로
		- 기각 = 문제 없다고 판단, 신고만 닫음
	- 신고가 들어와도 자동으로 숨기지 않음. 판단은 항상 운영자가 함

## 공통 규칙

- 삭제, 차단 같은 버튼은 무조건 "정말 하시겠어요?" 확인창을 한 번 거침
- 진짜 삭제(데이터 파기)는 없음. 전부 표시만 바꾸는 방식
- 상태 배지는 전부 무채색 (공개·정상 = 회색, 숨김·차단·미처리 = 검정, 삭제·탈퇴 = 희미)
	- 키컬러는 활성 메뉴와 "미처리 신고" 경고 숫자에만 씀

## 이번에 일부러 뺀 것

관리자 초대 화면, 권한 등급, 공지사항·이벤트 관리, 통계 그래프

→ 뺀 이유: 어드민의 최소 책무는 "보이는 걸 정리할 수 있다"까지. 그 이상은 운영하다가 실제로 아파지면 그때 추가.
