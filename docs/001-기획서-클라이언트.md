# 베이스보드 - 유저 사이드 기획

> 이 문서는 본보기 템플릿. "게시판"을 자기 아이템(상품, 장소, 레시피 등)으로 바꿔 쓰면 됨.

## 한 줄 정의

누구나 읽을 수 있고, 로그인한 사람만 쓸 수 있는 게시판 앱.

## 핵심 규칙 (이거 하나가 전체 구조를 결정함)

- 읽기: 로그인 없이 전부 가능
- 쓰기(글, 댓글, 좋아요): 구글 로그인 필요
	- 로그인 안 한 사람이 쓰기를 시도하면 → 그 자리에서 로그인 팝업
	- 로그인 끝나면 하려던 행동으로 그대로 복귀

## 로그인 = "시작하기"

- 유저에게 보여줄 땐 "로그인/회원가입"이라는 말 대신 "시작하기"로 통일
	- 버튼 문구: "Google로 시작하기"
	- 비로그인 마이 탭 = 시작하기 화면 (서비스 소개 한 줄 + 시작 버튼)
	- 글을 쓰려고 할 때 뜨는 안내 창도 같은 콘셉트
- 구글 로그인 딱 하나만. 이메일 가입, 전화번호 인증 없음
- 처음 로그인하면 그게 곧 가입. 가입 폼, 온보딩 화면 없음
- 닉네임은 자동 생성 (예: "졸린 쿠키") → 마이페이지에서 바꿀 수 있음
- 프로필 사진은 구글 계정 사진 그대로 사용
- 로그인 버튼 아래에 "계속하면 이용약관·개인정보처리방침에 동의하게 됩니다" 문구 (별도 동의 체크 화면 없음)
- 탈퇴하면 쓴 글은 남고 작성자가 "탈퇴한 사용자"로 바뀜
	- 탈퇴했던 계정이 다시 로그인하면 재가입으로 살아남

## 화면 구조

하단 탭 2개: 홈 / 마이

- 홈 = 글 목록
	- 목록 한 줄에: 제목, 닉네임, 시간("3시간 전"), 좋아요 수, 댓글 수
	- 최신순 하나만. 검색, 인기순 없음
	- 우하단에 동그란 글쓰기 버튼 (로그인한 사람만 동작)
- 글 상세
	- 제목, 닉네임, 시간, 본문, 좋아요 버튼
	- 아래에 댓글이 시간순으로
	- 우상단에 ⋯ 메뉴: 내 글이면 수정/삭제, 남의 글이면 신고
	- 댓글마다 ⋯ 메뉴: 내 댓글이면 삭제, 남의 댓글이면 신고
- 글쓰기 / 글수정
	- 제목 한 줄 + 본문. 끝. (임시저장, 이미지 첨부 없음)
	- 수정하면 글에 "수정됨" 표시
- 마이
	- 프로필 사진, 닉네임 (수정 가능)
	- 내가 쓴 글 목록
	- 설정: 마케팅 수신 동의 토글 (언제든 켜고 끌 수 있음)
	- 로그아웃, 회원탈퇴
	- 맨 아래 푸터: 이용약관, 개인정보처리방침, 문의 메일, © 표기
	- 로그인 안 하고 들어오면 → 로그인 안내 화면
- 이용약관 / 개인정보처리방침
	- 각각 별도 페이지. 푸터와 로그인 문구에서 진입
- 없는 주소로 들어오면 → "페이지가 없어요" 안내 + 홈으로 가기 버튼

## 글 · 댓글 · 좋아요 규칙

- 글: 본인 글만 수정·삭제 가능. 삭제해도 데이터는 보관 (화면에서만 사라짐)
- 댓글: 텍스트만, 대댓글 없음. 본인 댓글 삭제만 가능 (수정은 없음 - 지우고 다시 쓰기)
	- 삭제된 댓글 자리는 "삭제된 댓글입니다"로 표시
- 좋아요: 글마다 1인 1회, 다시 누르면 취소
- 신고: 남의 글·댓글의 ⋯ 메뉴에서 신고 → 사유 선택(스팸·홍보 / 욕설·비방 / 음란물 / 기타) → 접수
	- 접수되면 운영자(어드민)에게 전달됨. 신고했다고 글이 바로 사라지진 않음
	- 같은 사람이 같은 대상을 또 신고하면 기존 접수로 갈음

## 이번에 일부러 뺀 것

이미지 업로드, 검색, 카테고리(게시판은 1개), 대댓글, 알림, 인기순, 프로필 사진 변경

→ 뺀 이유: 베이스라인은 "로그인 규칙 + 글쓰고 반응하기"가 끝까지 돌아가는 게 목적. 위 기능들은 전부 이 골격 위에 나중에 얹는 살.

---

구현 전제 (기획과 무관): Next.js + Supabase(구글 로그인, DB) + Vercel 배포
- 1단계(현재): 진짜 로그인·DB 없이 만듦. 구글 버튼은 누르면 가상 유저 1명("졸린 쿠키")으로 즉시 로그인되는 가짜 버튼. 데이터는 서버 메모리에 미리 넣어둔 예시 데이터(글 10개 + 댓글·좋아요)로 동작하고, 화면과 데이터가 오가는 통로(API)만 진짜로 갖춰서 나중에 진짜 DB로 갈아끼움
- 화면 기준: 360×800 모바일 프레임 (디자인 시스템 문서의 "화면 틀" 참조)
